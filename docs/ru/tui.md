# Telegram TUI

OpenLinear рендерит все Telegram-страницы как **rich-сообщения** (Bot API 10.1): страница — это HTML-тело, отправляемое через `sendRichMessage` / `editMessageText` с полем `rich_message`. Telegram раскладывает его нативно — заголовки (`<h4>`/`<h5>`), таблицы (`<table>`), цитаты (`<blockquote>`), списки (`<ul>/<li>`) и раскрывашки (`<details>`).

## Раскладка

Фиксированной ширины больше нет — переносами занимается сам Telegram под клиента. Переносы строк внутри блока делаются через `<br>`, структура — через блочные теги выше. Дополнительно рендерер отдаёт текстовую версию того же контента (`Page.Text`) для CLI-предпросмотра (`openlinear render`) и тестов.

## Модель Страниц

OpenLinear считает одно Telegram-сообщение контейнером страниц.

Страницы:

- `main`
- `menu`
- страница категории
- детальная страница задачи
- выбор проекта (`/projects` или кнопка `📁 Projects`)
- страница проекта: прогресс и открытая работа по одному проекту

Проекты берутся из `projects.json`; пикер использует `id` проекта в
`callback_data`, а страница проекта фильтрует задачи по `name` проекта.

Навигация редактирует то же самое Telegram-сообщение через `editMessageText`.

## Главная Страница

Главная страница остаётся минимальной:

- шапка
- прогресс по статусам
- текущая работа
- review
- короткий next-up preview
- сводка attention
- счётчики скрытых разделов

Inline-кнопки:

- `Refresh`
- `Menu`

Кнопки конкретных задач живут внутри страниц категорий, а не на главной.

## Меню

Меню генерируется из `categories` в `settings.json`. Жёстко прошитого layout под backlog/next/attention нет.

## Страницы Категорий

Страницы категорий показывают все задачи, подходящие под фильтр категории. Если задач много, включается пагинация. Каждая задача получает компактную кнопку-плитку и короткую текстовую строку.

## Страница Задачи

Страница задачи показывает заголовок, описание в раскрывающемся `<details>`, метаданные таблицей и активность. Если у задачи есть `url`, появляется внешняя кнопка.

## Правила Безопасного Рендера

- Отправлять rich-HTML через `rich_message` (`sendRichMessage` / `editMessageText`).
- Экранировать динамический текст (`html.EscapeString`); переносы внутри блока — `<br>`.
- Не больше трёх inline-кнопок в ряд.
- `callback_data` ≤ 64 байт; тело сообщения в пределах rich-лимитов (32768 символов).
- Валидировать страницы через `ValidatePage` перед отправкой.
