# Telegram TUI

OpenLinear рендерит все Telegram-страницы как текст фиксированной ширины внутри HTML-блока `<pre>`. Это сохраняет выравнивание в разных клиентах Telegram.

## Ширина

Ширина по умолчанию — `30` ячеек. Она специально выбрана узкой для мобильных экранов.

Все операции рендера считают display-cell width, а не длину строки в байтах. Широкие символы считаются как две ячейки, combining marks — как ноль.

## Модель Страниц

OpenLinear считает одно Telegram-сообщение контейнером страниц.

Страницы:

- `main`
- `menu`
- страница категории
- детальная страница задачи

Навигация редактирует то же самое Telegram-сообщение через `editMessageText`.

## Главная Страница

Главная страница остаётся минимальной:

- шапка
- прогресс по статусам
- текущая работа
- review
- короткий next-up preview
- сводка attention
- счётчики скрытых разделов

Inline-кнопки:

- `Refresh`
- `Menu`

Кнопки конкретных задач живут внутри страниц категорий, а не на главной.

## Меню

Меню генерируется из `categories` в `settings.json`. Жёстко прошитого layout под backlog/next/attention нет.

## Страницы Категорий

Страницы категорий показывают все задачи, подходящие под фильтр категории. Если задач много, включается пагинация. Каждая задача получает компактную кнопку-плитку и короткую текстовую строку.

## Страница Задачи

Страница задачи показывает полный заголовок, многострочное описание, метаданные, активность и связи. Если у задачи есть `url`, появляется внешняя кнопка.

## Правила Безопасного Рендера

- Использовать `<pre>` с `parse_mode=HTML`.
- Держать строки в пределах настроенной ширины.
- Не использовать emoji в TUI-тексте.
- Делать не больше трёх inline-кнопок в ряд.
- Обрезать динамический контент по display width.
- Валидировать страницы перед отправкой.
